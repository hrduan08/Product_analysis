# YouTube & Reddit 反馈抓取 Demo 交互原型稿

## 1. 页面结构概览
- 核心流程逐步包含：登陆/注册 → 订阅管理 → 套餐购买 → 数据订阅 → 推送邮件 → 退订。
- 主要界面新增：登录页、注册页、邮箱验证提醒页、仪表盘（订阅列表 + 套餐提示）、创建订阅弹窗、套餐升级页、支付中/成功/取消页、账单历史页、退订确认页。

```
┌───────────────────────────┐            ┌────────────────────────┐
│ 登录 / 注册 页面          │            │ 订阅仪表盘 Dashboard   │
│  - 登录/注册切换 Tab      │  登录后 →   │  - 试用 / 套餐提示    │
│  - 第三方登录占位         │            │  - 订阅列表 + 创建按钮 │
└───────────────────────────┘            └────────────────────────┘
                                             │
                       ┌─────────────────────▼────────────────────┐
                       │ 创建订阅 Modal                           │
                       │  - 输入关键词 (最多 N 个)               │
                       │  - 选择平台（多选）                     │
                       │  - 频率（日/自定义）                    │
                       │  - 添加收件邮箱                         │
                       └─────────────────────────────┬────────────┘
                                                     │ 保存后
                       ┌─────────────────────▼────────────────────┐
                       │ 套餐升级页 Checkout                      │
                       │  - 套餐对比表                            │
                       │  - Stripe Checkout 按钮 / 微信支付       │
                       └──────────────────────────────────────────┘
```

## 2. 登录与注册
- **登录页**：
  - 左侧展示品牌介绍 / 产品价值，右侧登录表单。
  - 表单：邮箱、密码、记住我、登录按钮；下方提供“忘记密码”“注册新账号”链接。
  - 第三方登录按钮（Google/Apple）占位，后续可启用。
  - 错误提示区域：登录失败时显示“邮箱或密码错误”。
- **注册页**：
  - 与登录页同结构，改为“注册”按钮；新增密码强度提示、勾选《服务条款》和《隐私政策》。
  - 注册成功后跳转至“邮箱验证提醒页”。
- **邮箱验证提醒页**：
  - 顶部提示“验证您邮箱以开始试用”；显示已发送到哪个邮箱，并提供“重新发送邮件”“修改邮箱”按钮。
  - 支持直接跳转至登录页。

## 3. 订阅仪表盘（Dashboard）
- **头部信息栏**：
  - 左侧显示试用剩余天数或当前套餐信息（例如“免费试用还剩 5 天”/“Pro 套餐有效期至 2025-01-01”）。
  - 右侧提供“升级套餐”按钮（跳转套餐页）。
- **订阅列表**：
  - 卡片形式列出每条订阅：关键词列表、平台标签、频率、收件邮箱、下次执行时间、最近发送概览（新增/发送成功、本次邮件 ID）。
  - 卡片右侧提供操作：编辑、暂停/恢复、删除。
- **创建订阅 CTA**：
  - 页面右上角按钮“新建订阅”；试用/套餐配额不足时按钮禁用并提示“已达到最大订阅数”。
- **配额提示**：
  - 在列表顶部展示用量概览，如“剩余关键词 5/10、剩余订阅 2/5”。

## 4. 创建/编辑订阅弹窗
- 多步表单（Step Form）：
  1. **基础信息**：订阅名称（可选）、说明文字。
  2. **关键词设置**：Tag 输入，自带去重、最大数量提示、示例关键词。
  3. **平台选择**：复选框（YouTube、Reddit），显示可选平台数量限制。
  4. **推送频率**：选择每日固定时刻（如 9/12/15/18）或每周固定某天；若套餐允许自定义 Cron，提供高级选项。
  5. **收件邮箱**：列表输入，默认注册邮箱，可额外添加，需校验格式。
- 弹窗底部：上一步/下一步按钮及提交按钮；提交时显示 loading 状态。

## 5. 套餐与支付页面
- **套餐比较页**：
  - 表格列出“试用”“Pro 月付”“Pro 年付”等；对比关键词上限、订阅上限、邮件频率、支持平台等。
  - 每个套餐下方“立即升级”按钮。Pro 方案按钮触发 Stripe Checkout；国内支付弹出二维码模态。
  - 页面顶部显示当前套餐状态与剩余时间。
- **支付流程**：
  - Stripe：点击后跳转至 Stripe Checkout（新窗口或 iframe）；支付成功/取消后的回跳页面显示状态与“返回仪表盘”按钮。
  - 微信/支付宝（预留）：弹出二维码，底部添加“已完成支付”确认按钮。
- **订单历史**：
  - 表格展示支付时间、套餐名称、金额、状态（成功/失败/退款）及操作（下载发票、查看详情）。

## 6. 邮件模板与通知
- **邮件模板示例**：试用即将到期、试用已到期、升级成功、支付失败、退订成功。
- **退订页面**：
  - 从邮件中的退订链接进入，显示订阅名称、关键词列表、平台，确认退订后提示“已退订，后续不再发送”。
- **状态通知**：
  - 如果订阅暂停或因套餐过期停用，在仪表盘卡片上显示黄色警告；同时发送提醒邮件。

## 7. 账号设置
- **账号信息页**：
  - 显示邮箱、姓名、时区；支持修改时区、语言。
  - 提供“修改密码”“退出所有设备”功能。
- **API Token（可选）**：
  - 展示/生成 API 令牌，方便后续开放 API 使用；列出已生成的 token 及失效时间。

## 8. 交互细节 & 状态说明
- 所有表单提交均使用 inline 校验（实时提示字段错误）。
- 长操作（抓取、支付）使用全屏/局部 Spinner，并支持取消。
- Toast / Banner 统一组件管理，区分成功、警告、错误类。
- 支持暗色模式（可选），至少保证卡片颜色对比度达标。
- Mobile 端：
  - 登录页上下布局，按钮占满屏宽。
  - 仪表盘将订阅卡片堆叠，操作改为底部抽屉。
